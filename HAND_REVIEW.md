# 핸드 자동 복기 — EV 손실 기반 실수(패착) 감지

> 2026-07-07 확정된 차기 핵심 기능. "이것 하나로 다른 포트폴리오와 구별되는" 1순위로 선정.
> **✅ 2026-07-07 구현 완료** — `review` 패키지(`HandReviewer.judge` 순수 판정 + 몬테카를로),
> `GET /api/tables/{id}/hands/{index}/review` · `GET /api/tables/{id}/review/session`,
> 프론트 "핸드 복기" 패널(프레임 되감기 + 타임라인 실수 마커 + 최대 실수 배너 + 세션 리포트).
> 판정 여유폭 MARGIN=0.02(몬테카를로 오차 흡수, 경계=팟오즈 정확히 일치 시 실수 아님).

## 왜 이 기능인가

**이미 있는 부품 3개(리플레이 + 이퀴티 엔진 + 스탯)를 조합만 하면 되는데, 결과물은 경쟁 서비스가 안 만드는 기능.**

- 리플레이(이벤트소싱 `HandLog`)로 핸드의 모든 의사결정 지점 재현 가능
- 몬테카를로 이퀴티로 각 지점의 "그때 이퀴티 vs 팟오즈" 계산 가능
- → 핸드 종료 후 **"턴에서 이퀴티 18%인데 팟오즈 33% 콜 → 이 핸드 최대 실수(-4.2bb EV)"** 자동 리포트

구별되는 이유 3가지:
1. **포트폴리오 서사 완성** — 바둑 프로젝트의 "패착 감지"와 동일 컨셉. 두 프로젝트가 "복기 문화를 게임에 심는 개발자"라는 하나의 이야기로 묶임. 면접에서 프로젝트들이 일관된 관심사로 읽힘.
2. **차별점 스토리와 정합** — RuleGuard가 "박아박아를 룰로 막는" 소극적 장치라면, 이건 "왜 손해였는지 보여줘 스스로 고치게 하는" 적극적 장치. "한국 웹보드는 잃게 만들고, 내 서비스는 늘게 만든다"는 README 한 줄이 나옴.
3. **경쟁 부재** — PokerNow도 핸드히스토리 텍스트만 제공, 분석 없음. 실시간 이퀴티 표시는 흔하지만 **핸드 종료 후 자동 실수 지적하는 홈게임 서비스는 사실상 없음**.

## 스코프 (터지지 않게)

GTO 흉내 금지. **수학적으로 반박 불가능한 것만**:
- 각 콜/폴드 지점: 실제 이퀴티 vs 팟오즈 비교 → EV 손실 수치화
- 핸드당 **"최대 실수 1개"만** 표시 (바둑 패착과 동일 UX)
- 세션 누적 리포트: "이번 세션 EV 손실 합계 -23bb, 최다 유형: 리버 오버콜"

**제외**: 레이즈 사이즈 적정성, 블러프 빈도 등 GTO 영역 — 손대면 늪.

## 구현 스케치

- `review` 패키지 신설. 입력: `HandLog`(완료 핸드) → 출력: `HandReview`(지점별 판정 + 최대 실수).
- 각 콜 지점에서: 당시 히어로 홀카드 + 보드로 몬테카를로 이퀴티(기존 `equity` 재사용, 상대 레인지는 랜덤 가정 명시) vs 팟오즈 → `EV손실 = (필요이퀴티 - 실제이퀴티) × 팟` 근사.
- 폴드 판정도 동일(이퀴티가 팟오즈 크게 상회하는데 폴드 = 실수).
- API: `GET /api/tables/{id}/hands/{handId}/review`. 프론트: 리플레이 화면에 실수 지점 마커.
- 완료기준: 판정 로직 단위테스트(경계: 정확히 팟오즈 = 실수 아님) + 리플레이 UI 마커 표시.

## 차점 아이디어 (2순위, 백로그)

**검증 가능한 셔플(commit-reveal)** — 핸드 시작 전 셔플 해시 공개, 종료 후 시드 공개로 조작 불가 증명. 한국 유저의 "조작 아니냐" 불신을 정면으로 치는 기능이고 구현도 작지만, 기존 부품 재사용이 없고 서사 연결이 약해 2순위.
