# 포커 홈게임 서버 — 포트폴리오 프로젝트 계획

## 정체성
"제대로 된 룰을 강제하는 실시간 한국어 홈게임 서버" (PokerNow식 사적 홈게임 컨셉).
분석툴이 아니라 **실시간 멀티플레이 홀덤 엔진 + 게임룰 차별점**이 코어.
평가 핵심: 실서비스 배포보다 "이해하고 만들었는가"(면접용).

## 확정 결정
- **기술 스택**: Spring Boot + WebSocket/STOMP + React (기존 Java/Spring 강점 활용)
- **포트폴리오 코어**: 게임 엔진 + 분석 오버레이 **둘 다** (게임이 토대, 분석은 얇은 읽기전용 오버레이)
- **GTO 솔버 연동은 제외** (유료·멀티웨이 미지원, ROI 최악)

## 아키텍처
```
[React SPA] --STOMP/WebSocket--> [Spring Boot]
  테이블 UI / 액션 버튼          ├─ GameEngine (상태머신, 순수 POJO 도메인)
  실시간 이퀴티 오버레이         ├─ TableSessionManager (방/좌석/턴)
                                 ├─ RuleGuard (바이인·재입장·리로드 제약)
                                 ├─ EquityService (핸드평가·이퀴티)
                                 └─ StatsService (VPIP/PFR/ROI 집계)
```
**철칙: GameEngine은 WebSocket/DB/React를 전혀 모르는 순수 도메인.** 단위테스트로 100% 검증 가능하게 → 동시성/상태관리 역량 증거.

## 모듈 결정
- **게임 엔진**: 상태머신 PREFLOP→FLOP→TURN→RIVER→SHOWDOWN, 각 스트리트 내 베팅 라운드. 이벤트 소싱(액션 리스트→상태 재현)으로 가면 리플레이/분석 오버레이 공짜. 동시성은 테이블당 단일 스레드(액션 큐)로 락 없이 순차 보장.
- **이퀴티 오버레이**: Java 7-card evaluator 임베딩 또는 몬테카를로 롤아웃 직접 구현(수만 회면 충분). 서버가 각 플레이어에게 "본인 이퀴티만" push(상대 카드 노출 방지).
- **룰 차별점(RuleGuard)** = 진짜 셀링포인트: 최소 바이인 강제 / 버스트 후 재입장 쿨다운 / 일일 리로드 한도. 정책으로 격리. "한국 웹보드의 박아박아를 코드로 막았다"를 README 스토리로.
- **프리플랍 레인지**: 정적 JSON 하드코딩(100bb 6-max 표준). API 불필요.

## 빌드 순서 (스코프 관리)
1. **게임 엔진 도메인 + 단위테스트** (WebSocket 없이 콘솔/테스트로 한 판 끝까지) ← 여기서 80% 승부
2. STOMP 연동 + React 테이블 UI (2인용부터)
3. RuleGuard 정책 3개
4. 이퀴티 오버레이 (몬테카를로)
5. StatsService + ROI 리더보드

## 개발 환경
- 로컬 PC: i7-13700F(16코어/24스레드), RAM 16GB, RTX 4060 — 무거운 연산은 로컬이 서버보다 나음
- 배포용 서버: AWS Lightsail 2GB(vCPU 1개) — 사양 낮음, 배포 시 고려
