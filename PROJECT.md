# PROJECT — 홈포커 (Home Poker)

> 제대로 된 룰을 강제하는 실시간 한국어 홈게임 포커 서버.
> "배포"가 아니라 **"이해하고 만들었는가"**에 초점을 둔 포트폴리오 프로젝트.

---

## 한 줄 정의

노리밋 텍사스 홀덤을 **순수 도메인 게임 엔진**으로 구현하고, 그 위에 STOMP/WebSocket
실시간 계층과 React 테이블 UI를 얇게 올린 홈게임 서버.

## 왜 만드는가 (평가 포인트)

- 이 프로젝트의 심사 기준은 실서비스 트래픽이 아니라 **"도메인을 이해하고 스스로 설계했는가"**(면접용)다.
- 그래서 화려한 인프라보다 **상태머신·동시성·리댁션** 같은 "제대로 짜야 티 나는" 부분에 힘을 준다.
- 분석툴이 코어가 아니다. **실시간 멀티플레이 홀덤 엔진 + 게임룰 차별점**이 코어이고,
  이퀴티/통계는 그 위에 얇게 얹은 읽기 전용 오버레이다.

## 차별점 (셀링 포인트)

**RuleGuard** — 한국 웹보드 포커의 '박아박아'(무제한 리바이·리로드로 판을 망치는 행태)를
**코드로 막는** 정책 계층.

- 최소 바이인 강제
- 버스트(파산) 후 재입장 쿨다운
- 일일 리로드 한도

정책을 게임 엔진에서 격리해, "룰을 강제하는 서버"라는 스토리를 코드로 증명한다.

## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 백엔드 | Spring Boot 4.1 / Java 21 | 기존 Java·Spring 강점 활용 |
| 실시간 | WebSocket + STOMP + SockJS | 방/좌석/턴 브로드캐스트 |
| 프론트 | React + Vite | 테이블 UI, 개인화 상태 구독 |
| 이퀴티 | 자체 몬테카를로 롤아웃 | 외부 솔버 미사용(유료·ROI 최악) |

> **제외 결정**: GTO 솔버 연동 — 유료·멀티웨이 미지원으로 투자 대비 효과 최악.

## 현재 상태 (로드맵)

1. ✅ 게임 엔진 도메인 + 단위테스트 (WebSocket 없이 콘솔/테스트로 한 판 끝까지)
2. ✅ STOMP 연동 + React 테이블 UI (2인용)
3. ✅ RuleGuard: 최소 바이인 / 재입장 쿨다운 / 일일 리로드 한도
4. ✅ 이퀴티 오버레이 (몬테카를로, 본인 이퀴티만 push)
5. ✅ StatsService + ROI 리더보드 (VPIP/PFR/net, `GET /api/leaderboard`)
6. ✅ 핸드 히스토리 리플레이 (이벤트 소싱 `HandLog`, `GET /api/tables/{id}/hands`)
7. ✅ 멀티 테이블 로비 (`GET /api/tables`) + 타임뱅크 (30초 제한·자동 액션)
8. ✅ 통계 영속화 (JSON 파일 스냅샷, 재시작 후 리더보드 복원)
9. ✅ 관전 모드 + 프리플랍 레인지 오버레이 (6-max RFI, `GET /api/preflop-range`)

전체 74개 단위·통합 테스트 통과. 다음 후보는 [TODO.md](TODO.md) 참고.
